# Enterprise AI Agent Platform 产品需求文档

## 1. 项目概述

### 1.1 项目名称

Enterprise AI Agent Platform

### 1.2 项目简介

本项目是一个面向企业知识检索和设备故障诊断场景的 AI Agent 平台。

用户可以上传设备说明书、维修手册、故障案例和运行日志。系统通过 RAG、工具调用和 Agent 工作流，结合企业知识库与设备数据，生成带引用来源的故障分析结果和处理建议。

### 1.3 项目目标

第一阶段完成一个可运行、可演示、可部署的物联网设备故障诊断 Agent。

项目需要体现以下能力：

- Python 后端开发
- REST API 设计
- 数据库存储
- 企业文档知识库
- RAG 检索增强生成
- 大模型工具调用
- Agent 工作流
- 日志记录与异常处理
- 自动化测试
- Docker 部署

---

## 2. 项目背景

企业设备维护过程中，设备手册、维修记录、故障案例和运行数据通常分散在不同文件和系统中。

当设备发生故障时，维护人员需要人工查阅文档、分析日志、查询设备状态，处理效率较低，也容易遗漏关键信息。

本项目通过 AI Agent 整合设备知识库和结构化设备数据，为维护人员提供统一的智能诊断入口。

## 4. 核心业务场景

### 4.1 故障知识问答

用户输入设备故障问题，例如：

> 设备出现 E101 报警，同时温度持续升高，可能是什么原因？

系统从设备说明书、维修手册和故障案例中检索相关内容，并生成带引用来源的回答。

### 4.2 设备状态查询

用户输入：

> 查询设备 DEV-001 最近的运行状态。

Agent 调用设备查询工具，从数据库中读取设备温度、电压、在线状态和最近报警记录。

### 4.3 综合故障诊断

用户输入故障现象后，Agent 自动完成：

1. 识别设备编号和故障类型。
2. 查询设备实时数据或历史数据。
3. 检索相关知识库文档。
4. 综合设备数据与文档内容。
5. 输出可能原因、风险等级和排查步骤。
6. 返回引用来源。

### 4.4 文档管理

管理员可以上传以下文件：

- PDF 设备说明书
- TXT 故障案例
- Markdown 维修文档

系统完成文档解析、文本切分、向量化和入库。

## 7. 核心业务流程

### 7.1 文档入库流程

1. 管理员上传文档。
2. 后端校验文件类型和文件大小。
3. 系统解析文档文本。
4. 系统将文本切分为多个文本块。
5. 系统生成文本向量。
6. 系统保存文本块、向量和文档元数据。
7. 文档进入可检索状态。

### 7.2 故障诊断流程

1. 用户提交故障问题。
2. 系统识别问题意图。
3. Agent 判断需要调用的工具。
4. Agent 查询设备数据库。
5. Agent 检索知识库。
6. 大模型综合工具执行结果。
7. 系统生成故障分析结果。
8. 系统返回原因、建议和引用来源。

---

## 8. 输出结果要求

故障诊断结果至少包含：

- 问题摘要
- 设备状态
- 可能故障原因
- 风险等级
- 推荐排查步骤
- 引用文档
- 免责声明

示例：

```json
{
  "device_id": "DEV-001",
  "risk_level": "high",
  "possible_causes": [
    "散热风扇异常",
    "温度传感器故障"
  ],
  "recommended_actions": [
    "检查风扇运行状态",
    "检查散热口是否堵塞",
    "重新校验温度传感器"
  ],
  "sources": [
    "设备维护手册，第 12 页"
  ]
}
```

## 11. 项目风险

### 11.1 技术风险

- 大模型输出结果可能不稳定
- RAG 检索结果可能不准确
- 文档解析可能失败
- Agent 工具调用可能发生错误
- 不同模型 API 的接口存在差异
- 模型 API 可能产生调用费用
- 向量模型和大模型可能需要额外网络环境

### 11.2 项目实施风险

- 第一阶段功能范围过大
- 学习内容过多导致项目延期
- 过度依赖 AI 生成代码
- 代码可以运行，但本人无法解释
- 同时使用过多框架增加调试难度
- 时间有限导致项目无法在秋招前完成

### 11.3 风险控制措施

- 优先完成最小可用版本
- 每次只实现一个明确功能
- 第一阶段不加入非必要功能
- 优先使用成熟、简单的技术方案
- 核心模块必须编写测试
- 每次 Git 提交保持较小改动
- 所有核心代码必须能够解释
- 每周进行一次功能验收
- 非核心功能统一放入后续版本
